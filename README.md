# To-Do List Application

Тестовое задание: To-Do List на Laravel 11 + Nuxt 3 (Vue 3).

## Описание
Приложение разделено на 2 части: `backend` и `frontend`. 
- **Backend**: Laravel, MySQL, Sanctum (Bearer Token).
- **Frontend**: Nuxt 3, Vue 3, Vanilla CSS (Premium Dark Theme).

## Выбранный Auth-подход
Использован **Sanctum API Tokens (Bearer Token)**. Данный подход лучше всего подходит для локального развертывания, так как он не требует строгой настройки CORS/доменов, в отличие от Cookie Session, и упрощает старт проекта на различных портах.

## Быстрый старт (Docker Compose)

Для проверки проекта "в один клик" подготовлена конфигурация Docker. Из корня проекта выполните:

```bash
docker-compose up --build -d
```

Эта команда автоматически:
1. Поднимет базу данных MySQL.
2. Поднимет Backend (установит зависимости, выполнит миграции и сиды, запустит API на порту 8000).
3. Поднимет Frontend (установит модули и запустит Nuxt на порту 3000).

API будет доступно по адресу `http://localhost:8000`, а Frontend — `http://localhost:3000`.

## Команды ручной установки и запуска (Без Docker)

### Backend (Laravel)

1. Перейдите в папку `backend/`:
```bash
cd backend
```
2. Установите зависимости (если еще не установлены):
```bash
composer install
```
3. Скопируйте файл `.env.example` в `.env` и сгенерируйте ключ (если скачиваете из Git):
```bash
cp .env.example .env
php artisan key:generate
```
4. Настройте базу данных в `.env` (по умолчанию настроен MySQL: база `laravel_todo`, пользователь `root`). Убедитесь, что MySQL запущен и база данных создана.
5. Выполните миграции и сиды:
```bash
php artisan migrate:fresh --seed
```
6. Запустите сервер:
```bash
php artisan serve
```

### Frontend (Nuxt 3)

1. Перейдите в папку `frontend/`:
```bash
cd frontend
```
2. Установите зависимости:
```bash
npm install
```
3. Скопируйте файл `.env.example` в `.env` в папке `frontend/`:
```bash
cp .env.example .env
```
4. Пропишите базовый URL API (по умолчанию сервер работает на порту 8000):
```env
NUXT_PUBLIC_API_BASE_URL=http://localhost:8000/api
```
5. Запустите dev-сервер:
```bash
npm run dev
```

## Документация API (Swagger)

Для API поднята **интерактивная документация Swagger UI** на базе пакета `l5-swagger`! 

Она генерируется автоматически прямо из **PHP Attributes** (`#[OA\Get]`, `#[OA\Post]` и т.д.) внутри контроллеров.

### Как сгенерировать / обновить документацию:
Если вы внесли изменения в параметры или добавили новый эндпоинт в контроллере, выполните команду в папке `backend/`:
```bash
php artisan l5-swagger:generate
```

### Где посмотреть:
Чтобы открыть визуальный интерфейс (убедитесь, что бэкенд запущен через `php artisan serve`), перейдите по ссылке:
👉 **http://localhost:8000/api/documentation**

Там вы сможете изучить все эндпоинты, посмотреть требуемые параметры и протестировать запросы прямо из браузера (не забудьте вставить токен в кнопку "Authorize").

### Краткое описание эндпоинтов

**Авторизация**
- `POST /api/auth/login` — Вход в систему (возвращает `access_token`).
- `POST /api/auth/logout` — Выход (отзывает токен).
- `GET /api/user` — Получение данных текущего авторизованного пользователя.

**Задачи (Требуют заголовок `Authorization: Bearer {token}`)**
- `GET /api/tasks` — Получить список задач (пагинированный).
  - *Query-параметры*: `search` (строка), `status` (pending|in_progress|completed), `sort` (поле), `dir` (asc|desc), `page` (число).
- `POST /api/tasks` — Создать новую задачу.
- `GET /api/tasks/{id}` — Получить информацию о конкретной задаче.
- `PUT /api/tasks/{id}` — Обновить задачу (пользователь может обновлять только свои задачи, админ — любые).
- `DELETE /api/tasks/{id}` — Удалить задачу (пользователь может удалять только свои задачи, админ — любые).

**Формат ошибок**
API всегда возвращает ошибки в стандартизированном JSON-формате:
- `401 Unauthorized` — если токен не передан или недействителен.
- `403 Forbidden` — если попытка доступа к чужой задаче.
- `404 Not Found` — если задача не найдена.
- `422 Unprocessable Entity` — при ошибках валидации входящих данных (возвращается объект `errors`).
- `500 Internal Server Error` — при непредвиденных ошибках сервера.
